# Tech Longlist Creator

技術調査リストに基づいて、企業・組織の技術情報を網羅的に収集し、構造化されたjsonファイル形式のロングリストを作成する。

## Overview

このスキルは、研究開発部門向けの技術調査プロセスを自動化する。
調査リスト（Excel形式）の各行について、WEB検索を実施し、公式サイトを中心とした信頼性の高い情報源から技術情報を抽出する。

## Workflow

### Step 1: 調査リストの読み込み

調査リストExcelファイルから以下の情報を取得:
- No: 調査対象ID
- 調査対象: 企業名・組織名
- 特記事項: 特に確認すべき内容
- 関連キーワード: 検索に使用するキーワード
- URL: 公式サイトのURL

### Step 2: 情報収集

各調査対象について以下の手順で情報を収集:

1. **公式サイトの確認**
   - 提供されたURLから公式サイトにアクセス
   - 製品・技術情報ページを特定

2. **WEB検索の実施**
   - 企業名・製品名で検索
   - 関連キーワードを組み合わせて検索
   - 公式ドメインおよびサブドメインの情報を優先

3. **情報源の選別**
   - 公式サイト内の情報のみを使用
   - 公式ニュースリリース、製品ページ、技術文書を優先
   - サードパーティのメディア記事は補助的に使用

### Step 3: 情報の構造化

収集した情報を以下の項目に整理:

**基本情報** (8項目)
- ID, タイトル, 概要1-4

**注目ポイント** (3項目)
- 注目ポイント1-3

**技術・情報詳細** (5項目)
- 技術開発の進展度, 主要情報ソースタイプ, 主要情報ソースURL, 発表年または出願年, 代表的な図のURL, 図の掲載ページURL

**組織情報** (5項目)
- 組織名, 組織タイプ, 国名, 組織設立年, 組織URL

**情報ソース** (20項目)
- 情報ソースURL 1-10（最優先）
- 情報ソースURL 11-20（補助的）

**分類または補足情報** (X項目)
- 技術リスト（Excelファイル）に記載されているフィールド名から判断し、必要分だけ追加する
- 例：技術分野、機能分類、製品カテゴリ、臨床試験フェーズ、価格、スペック、適用市場など目的により様々

## 基本情報項目

### ID
- **内容**: 調査リストのNo列の値をそのまま使用
- **形式**: 数値または文字列

### タイトル
- **文字数**: 120文字以内
- **内容**: 技術を特徴づける材料、デバイス、サービス、システムを含む
- **形式**: 技術名やサービス名で終わる体言止めの一文
- **表記ルール**:
  - 企業固有の技術名・サービス名は「」書き
  - 例: シリコンフォトニクスによる高集積センサーを用いたデジタル嗅覚ソリューション「NeOse Advance」
  - 商標:®やトレードマーク:™は記載しない
  - 開発組織名は含めない
  - サブタイトルは付けない
  - WEB検索結果の事実情報に基づくタイトルとする
- **例文**: インピーダンスセンサと加速度センサを用いてエネルギーバランスを自動的に追跡するスマートバンド「GoBe3」

### 概要1
- **文字数**: 150字程度
- **主語**: 技術を開発した組織名
- **内容**: 
  - 組織名は原文のまま、英語は英語表記のまま書く
  - 組織の所在国を文頭に簡潔に加える
  - 論文の場合、共同研究者に企業が含まれる場合は言及
- **末尾**: 「～を開発した。」「～を研究している。」「～を販売している。」などで終わる
- **例文**: スマートバンドの製造/販売を行う米国のHealbe Corporationは、生体インピーダンスセンサ・心拍センサ・加速度センサを搭載したリストバンド型デバイスと「HEALBE」アプリで、エネルギーバランス・体の水分レベル・ストレスレベルを自動追跡するスマートバンド「GoBe3」を開発/販売している。

### 概要2
- **文字数**: 250字以内
- **主語**: 「同技術 / 同サービス / 同デバイスは / 同研究では」と統一
- **内容**: 具体的な材料、デバイス、サービス、システムの原理、機能、構造、プロセスを説明。体言止めはせず完全な文章として作成。
- **注意**: 定量的な成果や特性は含めない
- **例文**: 同製品は、独自の生体インピーダンスセンサとアルゴリズムを用いて、体がエネルギーを吸収する際の細胞内液の排出レベルを計測して摂取カロリーを測定する。また、体重・身長・性別・年齢などのパラメータから基礎代謝カロリーと、内蔵された9軸加速度計（ジャイロスコープおよび磁力計）と光学式脈拍センサのデータから筋肉活動エネルギーを推定して消費カロリーを計測し、摂取/消費カロリーを非侵襲的かつ継続的に自動計測することでエネルギーバランスを監視する。

### 概要3
- **文字数**: 250字以内
- **主語**: 「同技術 / 同サービス / 同デバイスは、」と統一
- **内容**: 概要2の技術により得られた具体的な成果や特性を記載。体言止めはせず完全な文章として作成。
- **注意**: 定量的な成果や特性は含めない
- **例文**: 同製品は、摂取カロリーを高精度にトラッキングすることが可能である。これにより、エネルギーバランスを監視しながら効率的に筋肉量を増やしたり、ダイエットのためのトレーニングを設計できる。さらにスマートフォンとの連携により、リアルタイムなデータ確認や、アラート設定も可能である。

### 概要4
- **文字数**: 120字以内
- **内容**: 技術の用途として記載されている内容を簡潔にまとめる。
- **例文**: 同製品は、筋肉量を増やしたい人・体重を維持/減少したい人、また、健康を意識する消費者をターゲットとしている。

## 注目ポイント項目

### 注目ポイント1
- **文字数**: 130字×最大2文
- **内容**: 当該技術が直接解決しようとしている固有の課題や具体的な開発背景を記載
- **形式**: 箇条書き、文頭に「・(全角)」を使用
- **重要**: 業界全般の課題や一般的な課題は記載しない
- **例文**:・血液サンプルを要する侵襲的な血糖グルコース測定法は、利便性と安全性の面で課題があり、インピーダンスを測定する非侵襲的な方法は、測定精度が侵襲的手法より劣ることが課題となっていた。
・従来のインピーダンス測定には大型機器が必要であり、日常的に計測するには課題があった。

### 注目ポイント2
- **文字数**: 130字×最大3文
- **内容**: 技術の新規性、進歩性、優位性を簡潔に記載
- **形式**: 箇条書き、文頭に「・(全角)」を使用
- **例文**:・同製品は、臓器/組織での食物の消化/代謝過程の数学的モデルと検出された血中グルコース濃度の経時的変化に基づいて摂取カロリーを推定する。
・同製品は、高周波数と低周波数を用いたインピーダンスセンサを使用し、細胞内外への体液の動きを精密に測定する。
・同製品は、非侵襲的に摂取カロリー・⽔分バランス・ストレスレベル・睡眠の質などを⾃動計測できるスマートバンドである。

### 注目ポイント3
- **文字数**: 130字×最大2文
- **内容**: 客観的に優れていると認められたエビデンスを記載
  - 共同研究やパートナーシップの情報
  - 正確な発表時期が分かる場合は記載してもよい。
- **形式**: 箇条書き、文頭に「・(全角)」を使用
- **例文**: 同社は、同技術に関してApple, Inc.とのパートナーシップ契約を締結した（2025年3月）。

## 技術・情報詳細項目

### 技術開発の進展度
以下から該当するものを1つ選択:説明文は不要。LvX:〇〇　のみを記載

- **Lv1:アイデア** - 設計された機能を実現する理論的な裏付けはないアイデアである
- **Lv2:理論検証** - 新しい原理、理論が数学やシミュレーションなどにより検証されている
- **Lv3:実験段階** - 限定された理想環境下（実験室内）で、一部の機能について効果が検証されている
- **Lv4:試作段階** - 試作品が完成（仕様は未決定）し、より現実に近い環境下で、主要な機能の効果が検証されている
- **Lv5:製品検証** - 製品が完成（仕様は決定）し、全ての機能の効果が検証されている
- **Lv6:承認** - 製品品質が規定の基準をクリアし、規制当局から承認を得ている（医療機器や電気製品、通信機器など）
- **Lv7:販売・実用化** - 製品・技術が販売・上市、または実用化されている

## 基本原則

### 客観性の徹底
これは技術文書である。事実と根拠以外は書かず、以下を避ける:
- 形容詞・副詞
- 婉曲表現
- 感嘆詞
- マーケティングの誇張表現
- アイキャッチ的な表現

### 禁止表現リスト
以下の語句が含まれる出力は不正とみなす:
- 世界初
- 革新的
- 画期的
- 完全に
- 新時代の

## 文体と表現
### 公用文の書き方・ルールに従う

### である調の使用
全体を通して「である調」（常体）を使用する。

**良い例:**
- 同技術は、非侵襲的に血糖値をモニタリングすることを可能とした。
- 同製品は、小型、軽量、高感度を特徴としている。

**悪い例:**
- 同技術は、非侵襲的に血糖値をモニタリングすることができます。
- 同製品は、小型、軽量、高感度です。

### 体言止めの禁止
各項目内で体言止めを使用しない。

**良い例:**
- 同製品は、糖尿病患者を対象に、健康モニタリングへの適用を意図している。

**悪い例:**
- 同製品は、糖尿病患者を対象に、健康モニタリングへの適用。

### 括弧の使用
半角の括弧()は使用不可。全角の（ ）を使用する。

**良い例:**
- マッハツェンダー干渉計（Mach Zehnder Interferometers:MZI）

**悪い例:**
- マッハツェンダー干渉計(Mach Zehnder Interferometers:MZI)

### 数字と単位
数字と単位の間は半角スペースを空ける。

**良い例:**
- 1.1 mM
- 0.1～1,000 ppm
- 215×130×50 mm
- 400 g

**悪い例:**
- 1.1mM
- 0.1～1,000ppm

**例外**
パーセント表記は半角スペース不要。例） 1.1%

## 用語の扱い

### 専門用語
技術的な専門用語は初出時に簡単な説明を付ける。

**例:**
- マッハツェンダー干渉計は、1つの光源から分けた2つの平行光の間の位相差を測定する光学機器である。

### 略語
初めて出現した時にフルスペルを記載し、以降は略語を使用する。

**良い例:**
- 揮発性化合物（Volatile organic compound:VOC）を検出する。同プラットフォームは、同臭気センサアレイが取得した信号パターンから、個々のVOCを同定する。

**悪い例:**
- VOCを検出する。（初出時にフルスペルがない）

### 用語の統一
同じ概念に対して異なる表現を使用しない。

**良い例:**
- 同技術は...、同技術は...、同技術は...（一貫して「同技術」を使用）

**悪い例:**
- 同技術は...、この技術は...、その技術は...（表現が統一されていない）

### 組織名
組織名は日本語に翻訳せず、英語の正式名称（リーガルネーム）で書く。

**良い例:**
- Harvard University

**悪い例:**
- ハーバード大学

### 商標表記
®やトレードマーク:™の特殊文字表記は書かない。

**良い例:**
- OrganoReady Colon Organoid

**悪い例:**
- OrganoReady ® Colon Organoid

## 主語の統一

### 概要1
技術を開発した組織名を主語とする。

**例:**
- 香港のヘルスケアスタートアップ企業Add Care Limitedは、近赤外分光吸収、動的代謝熱モデルおよびAIアルゴリズムを用いた非侵襲的グルコースモニタリングウォッチ「DynasynQ」を販売している。

### 概要2、概要3
「同技術」「同サービス」「同デバイス」「同研究」などで主語を統一する。

**例:**
- 同製品は、近赤外光の複数の波長帯域セットを用いて...
- 同製品は、同スペクトルデータから、動的代謝熱モデルおよび...

### レビュー論文の場合
文章のつながりに違和感がないように、文頭表現を最適化してもよい。
- 「同技術は～」
- 「同研究は～」
- 「同レビューでは～」

## ハルシネーション防止

### 情報源の明示性
情報ソースが明記している事実だけに基づいて書く。

### 推測の禁止
以下は記載しない:
- あなたの推測
- 拡大解釈
- 一般論
- 確認できない情報

### 根拠の確認
各文について、以下を確認:
- この情報は情報ソースに明記されているか？
- 推測や拡大解釈を含んでいないか？
- 一般的な知識を勝手に追加していないか？

## URLリンクの扱い

本文中にラベルを用いたURLリンクは作成しない。

**悪い例:**
- マクドナルドの情報を参照して作成した文章に、[マクドナルド](https://www.mcdonalds.com/)のような表現

**良い例:**
- 指定した形式の参照URL記述法のみで対処（情報ソースURL欄に記載）

## 文章構成の例

### 概要2の構成例
```
同製品は、[検出方法]を用いて、[対象]から[データ]を収集する。同製品は、同[データ]から、[処理方法]を用いて、[結果]を算出する。また、同製品は、[付加機能]を実現する。同製品による[測定]は、[制約条件]の影響を受けるため、[対処方法]を要する。
```

### 注目ポイントの構成例
```
・従来の[方法]は、[問題点]が課題であった。
・同[技術]は、[特徴1]、[特徴2]、[特徴3]を特徴としている。
・同[技術]は、[認証/受賞]を取得している。
```

## 品質チェックリスト

文章作成後、以下を確認:

- [ ] である調で統一されているか
- [ ] 体言止めを使用していないか
- [ ] 全角括弧（）を使用しているか
- [ ] 数字と単位の間に半角スペースがあるか
- [ ] 専門用語に説明を付けているか
- [ ] 略語を初出時に定義しているか
- [ ] 用語が統一されているか
- [ ] 組織名を英語で記載しているか
- [ ] 商標記号を除去しているか
- [ ] 主語が統一されているか
- [ ] 禁止表現を含んでいないか
- [ ] 情報ソースに基づく事実のみを記載しているか
- [ ] 推測や一般論を含んでいないか
- [ ] URLリンクを本文中に含めていないか
- [ ] 文字数制限を守っているか


### Step 5: JSON出力

収集・構造化した情報を1件ずつJSONファイルとして順次出力。各JSONファイルが1つの調査対象に対応。

## Usage Example

```python

import pandas as pd
import json

df = pd.read_excel('調査リスト.xlsx')

# 各行について情報収集・構造化・JSON出力を実行
for idx, row in df.iterrows():
    # 情報収集と構造化
    result = process_target(row)

    # JSONファイルとして即座に出力
    filename = f"{result['基本情報']['ID']}_{result['組織情報']['組織名']}.json"

        json.dump(result, f, ensure_ascii=False, indent=2)
```

## Key Points

### 情報源の優先順位

1. 公式サイトの製品・技術ページ
2. 公式ニュースリリース
3. 公式技術文書・ホワイトペーパー
4. 学術論文・特許文献（該当する場合）
5. 信頼性の高いメディアの記事（補助的に）

### 文字数制限

各項目には文字数制限あり。:
- タイトル: 120文字以内
- 概要1: 150字程度
- 概要2: 250字以内
- 概要3: 250字以内
- 概要4: 120字以内
- 注目ポイント1: 130字×最大2文
- 注目ポイント2: 130字×最大3文
- 注目ポイント3: 130字×最大2文

### 禁止表現

以下の表現は使用禁止（マーケティング的表現を避ける）:
- 世界初
- 革新的
- 画期的
- 完全に
- 新時代の

### ハルシネーション防止

- 情報ソースに明記されている事実のみを記載
- 推測や拡大解釈を行わない
- 一般論を含めない
- 確認できない情報は記載しない

## Notes

- このスキルは大量のWEB検索を伴うため、処理に時間がかかる。各調査対象の情報収集が完了次第、JSONファイルを即座に出力する。
- 公式サイトにアクセスできない場合、その旨を記録して次の対象に進む
- 情報が不足している場合、可能な範囲で記載し、不明な項目は"不明"とする
- JSON出力により、処理の途中経過が保存され、エラー発生時も完了済みのデータが失われない
